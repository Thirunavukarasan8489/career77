1. Purpose

The Candidate Resume feature allows Candidates to create, upload, manage, and maintain their professional resume.

The Resume is used for:

Job Applications
Recruiter Review
Candidate Profile
Job Matching

2. Resume Access

Route:

/candidate/resume

Access requires:

Authenticated
+
Role = Candidate
+
Account Status = Active
3. Resume Structure
Candidate Resume
│
├── Resume Overview
├── Upload Resume
├── Create Resume
├── Resume Preview
├── Download Resume
└── Resume Management
4. Upload Resume

The Candidate can upload an existing resume.

Supported formats should include:

PDF
DOC
DOCX

Upload flow:

Candidate
    ↓
Resume Page
    ↓
Upload Resume
    ↓
Select File
    ↓
Validate File
    ↓
Upload
    ↓
Resume Saved
5. Resume Validation

The system must validate:

File type.
File size.
File name.
File content where applicable.

Invalid files should display a clear error.

Example:

Invalid file format.

Please upload a PDF or DOCX file.
6. Create Resume

Candidates may create a resume using Career77 profile information.

Candidate Profile
    ↓
Resume Builder
    ↓
Select Template
    ↓
Review Information
    ↓
Customize Resume
    ↓
Save Resume

The Resume Builder may use:

Personal Information
Professional Summary
Skills
Experience
Education
Certifications
Projects
Languages
7. Resume Templates

Career77 may provide multiple professional resume templates.

Example:

Modern
Professional
Minimal
Technical
Executive

Templates should be:

Professional.
ATS-friendly.
Easy to read.
Responsive in preview.
Suitable for job applications.
8. Resume Preview

Candidates should be able to preview their resume before downloading or applying.

Resume Editor
    ↓
Preview
    ↓
Review Resume
    ↓
Save / Download

The preview should closely match the final exported document.

9. Download Resume

Candidates can download their resume.

Resume
    ↓
Download
    ↓
Generate File
    ↓
Download Resume

The downloaded resume should maintain the selected template and formatting.

10. Replace Resume

Candidates can replace their uploaded resume.

Existing Resume
    ↓
Replace
    ↓
Select New Resume
    ↓
Validate
    ↓
Upload
    ↓
New Resume Active

The previous resume should no longer be used as the default resume.

11. Delete Resume

Candidates can delete their uploaded resume.

Resume
    ↓
Delete
    ↓
Confirmation
    ↓
Delete Resume

Confirmation message:

Are you sure you want to delete this resume?

The system should prevent accidental deletion.

12. Primary Resume

Candidates may have multiple resumes.

Example:

Resume 1 — Full Stack Developer
Resume 2 — Frontend Developer
Resume 3 — Backend Developer

One resume can be marked as:

Primary Resume

The Primary Resume should be selected by default during job applications.

13. Resume Selection During Application

When applying for a job:

Job Detail
    ↓
Apply
    ↓
Select Resume
    │
    ├── Primary Resume
    ├── Other Resume
    └── Upload New Resume
    ↓
Submit Application

The Candidate should be able to select the most relevant resume.

14. Resume Status

The Resume may have the following states:

No Resume
Uploaded
Processing
Ready
Failed

Example:

Uploaded
    ↓
Processing
    ↓
Ready

If processing fails:

Processing
    ↓
Failed
    ↓
Retry / Upload Again
15. Resume Parsing

Career77 may extract information from uploaded resumes.

Possible extracted information:

Name
Email
Phone
Skills
Experience
Education
Certifications
Projects

The extracted information can be used to help complete the Candidate Profile.

16. Resume Data Review

If resume parsing is used:

Upload Resume
    ↓
Extract Information
    ↓
Show Extracted Data
    ↓
Candidate Reviews
    ↓
Candidate Confirms
    ↓
Update Profile

Candidates must be able to review and correct extracted information.


18. Resume Security

Resume files contain personal information and must be protected.

The system must:

Require authentication.
Verify Candidate ownership.
Restrict unauthorized downloads.
Secure uploaded files.
Validate file types.
Validate file sizes.
Protect resume URLs.
Prevent public access to private resumes.
19. Resume Storage

Resume files should be stored using secure file storage.

Possible storage:

Object Storage
    ↓
Secure File URL
    ↓
Authorized Candidate / Recruiter Access

The exact storage architecture must follow:

docs/08-architecture/file-storage.md
20. Resume Loading State

While loading:

Resume
    ↓
Loading
    ↓
Skeleton UI
    ↓
Resume Data

The interface should remain stable during loading.

21. Resume Empty State

If the Candidate has no resume:

No resume uploaded yet.

Upload an existing resume or create one with Career77.

Actions:

[ Upload Resume ]

[ Create Resume ]
22. Resume Error State

If the resume cannot be loaded:

Unable to load your resume.

Please try again.

Action:

[ Retry ]

Internal server errors must not be displayed to the Candidate.

    ↓

25. Resume Acceptance Criteria

The Resume feature is complete when:

Candidate can upload a resume.
Candidate can create a resume.
Candidate can preview a resume.
Candidate can download a resume.
Candidate can replace a resume.
Candidate can delete a resume.
Candidate can manage multiple resumes.
Candidate can select a Primary Resume.
Candidate can select a resume during application.
Uploaded files are validated.
Resume files are securely stored.
Unauthorized users cannot access private resumes.
Resume parsing works where implemented.
Candidates can review parsed information.

Resume loading states are handled.
Resume error states are handled.
Resume empty states are handled.
Resume feature is responsive.
Resume feature is accessible.
