1. Purpose

The Candidate Job Search feature allows Candidates to discover and find relevant job opportunities on Career77.

Candidates can:

Search Jobs
Filter Jobs
Sort Results
View Job Details
Save Jobs
Apply for Jobs
2. Job Search Access

Route:

/candidate/job-search

Access:

Authenticated Candidate

Public users may also search jobs through the public Job Search page.

3. Job Search Flow
Candidate
    ↓
Search Jobs
    ↓
Enter Keywords
    ↓
Apply Filters
    ↓
View Results
    ↓
Open Job Detail
    ↓
Apply / Save Job

Candidates can sort results by:

Most Relevant
Newest
Oldest

The default sorting should prioritize relevance.

If many jobs are available, results should support pagination.

Page 1
Page 2
Page 3

Alternatively, infinite scrolling may be used if approved by the product design.

10. No Search Results

If no jobs match the search:

No jobs found.

Try changing your search or filters.

Suggested actions:

Clear Filters
Search Again
View All Jobs
11. Saved Jobs

Candidates can save interesting jobs.

Flow:

Job
    ↓
Save Job
    ↓
Saved

Candidates can access saved jobs from their Candidate Portal.

Example:

Saved Jobs
    │
    ├── Frontend Developer
    ├── Backend Developer
    └── Full Stack Developer
12. Remove Saved Job

Candidates can remove a saved job.

Saved Job
    ↓
Remove
    ↓
Job Removed from Saved List

The system should update the saved state immediately.

13. Job Recommendation

Recommendation inputs may include:

Skills
Experience

Location

Previous Applications
Search Activity

Example:

14. Job Detail Navigation

The Candidate can select a job from search results.

Job Search
    ↓
Select Job
    ↓
Job Detail

The Job Detail page should provide complete information about the opportunity.

15. Apply from Search

Candidates may apply directly from search results.

Job Search
    ↓
Apply
    ↓
Authentication Check
    ↓
Select Resume
    ↓
Submit Application

If the Candidate is not authenticated:

Apply
    ↓
Login / Register
    ↓
Continue Application
17. Search URL Parameters

Search filters should be reflected in the URL where appropriate.

Example:

/candidate/job-search?keyword=react&location=coimbatore&experience=mid-level

This allows Candidates to:

Share searches.
Bookmark searches.
Refresh without losing filters.
Navigate using browser history.
18. Search Loading State

While loading results:

Search
    ↓
Loading
    ↓
Job Skeleton Cards
    ↓
Search Results

The interface should avoid unnecessary layout shifts.

19. Search Error State

If the search request fails:

Unable to load jobs.

Please try again.

Action:

[ Retry ]

Internal API or database errors must not be shown to Candidates.

21. Search Security

The system must:

Validate search parameters.
Protect authenticated Candidate data.
Prevent unauthorized saved-job access.
Prevent unauthorized application access.
Sanitize search inputs.
Enforce backend authorization.
22. Job Search Acceptance Criteria

The Job Search feature is complete when:

Candidates can search jobs.
Candidates can filter jobs.
Candidates can sort jobs.
Candidates can view job results.
Candidates can open Job Details.
Candidates can save jobs.
Candidates can remove saved jobs.
Candidates can apply for jobs.
Search filters are preserved where appropriate.
Search supports pagination or approved infinite scrolling.
Recommended jobs can be displayed.
Empty states are handled.
Loading states are handled.
Error states are handled.
Search is responsive.
Search is accessible.
Candidate data is protected.
