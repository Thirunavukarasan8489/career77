1. Purpose

The Candidate Portal provides Candidates with all the tools required to manage their career journey on Career77.

The Candidate Portal includes:

Dashboard
Profile
Resume
Job Search
Applications
Interviews
Settings

3. Candidate Journey

The main Candidate journey is:

Register / Login
      ↓
Complete Profile
      ↓
Create Resume
      ↓

      ↓
View Job
      ↓
Apply
      ↓
Track Application
      ↓

      ↓

4. Candidate Dashboard

The Dashboard provides a quick overview of the Candidate's activity.

Possible sections:

Dashboard
│
├── Profile Completion
├── Recommended Jobs
├── Recent Applications
├── Upcoming Interviews
└── Quick Actions
5. Candidate Profile

The Candidate can create and manage their professional profile.

Profile information may include:

Personal Information
Professional Summary
Skills
Experience
Education
Certifications
Projects
Languages
Location
Career Preferences

The profile should help recruiters understand the Candidate's background.

6. Resume

Candidates can manage their resume.

Possible features:

Upload Resume
View Resume
Download Resume
Replace Resume
Delete Resume

The Candidate should be able to maintain an up-to-date resume.

7. Job Search

Candidates can search for available jobs.

Search options may include:

Keyword
Job Title
Skills
Location
Experience Level
Employment Type

The Candidate can:

Search
    ↓
View Results
    ↓
Open Job Detail
    ↓
Apply
8. Job Application

The Candidate can apply for suitable jobs.

Job Detail
    ↓
Apply
    ↓
Candidate Authentication
    ↓
Application Form
    ↓
Submit Application
    ↓
Application Created

The system should prevent duplicate applications for the same job unless explicitly allowed.

9. Applications

Candidates can view their submitted applications.

Application information may include:

Job Title
Company
Applied Date
Application Status
Last Updated

Possible application statuses:

Applied
Under Review
Shortlisted
Interview
Selected
Rejected
Withdrawn
10. Application Tracking

The Candidate should be able to track the progress of each application.

Example:

Applied
   ↓
Under Review
   ↓
Shortlisted
   ↓
Interview
   ↓
Selected

The Candidate should receive appropriate updates when the application status changes.

11. Interviews

Candidates can view scheduled interviews.

Interview information may include:

Job
Company
Interview Date
Interview Time
Interview Type
Interview Location / Meeting Link
Interview Status

Example:

Upcoming Interview
    ↓
View Details
    ↓
Attend Interview
12. Notifications

Candidates may receive notifications for:

Application Updates
Interview Invitations
Interview Changes
Recruiter Messages
System Notifications

Notifications should be accessible from the Candidate Portal.

13. Candidate Settings

Candidates can manage account preferences.

Settings may include:

Account Information
Password
Email Preferences
Notification Preferences
Privacy
Account Deactivation

Sensitive account changes should require appropriate security verification.

14. Profile Completion

The system may calculate Candidate profile completion.

Example:

The system can suggest missing information such as:

Add Professional Summary
Add Skills

The goal is to encourage Candidates to maintain complete profiles.

15. Candidate Quick Actions

The Dashboard may provide quick actions:

Search Jobs
Update Profile
Upload Resume
View Applications
View Interviews

These actions should provide quick access to common Candidate tasks.

16. Candidate Access

Candidate Portal access requires:

Authenticated
+
Role = Candidate
+
Account Status = Active

If the Candidate is not authenticated:

Candidate Portal
    ↓
Login Required
    ↓
Candidate Login
17. Candidate Data Privacy

Candidate private information must not be publicly exposed.

Private information may include:

Email
Phone Number
Resume
Applications
Interview Details
Private Messages
Account Settings

Access to Candidate information must follow platform authorization rules.

18. Candidate Portal Navigation

Recommended navigation:

Career77
│
├── Dashboard
├── Profile
├── Resume
├── Find Jobs
├── Applications
├── Interviews
└── Settings

The navigation should be consistent across Candidate Portal pages.

19. Responsive Design

The Candidate Portal must support:

Desktop
Tablet
Mobile

Desktop:

Sidebar
    +
Main Content

Mobile:

Top Header
    +
Mobile Navigation
    +
Main Content

All Candidate features must remain usable on smaller screens.

20. Candidate Portal Security

The system must:

Require authentication.
Verify Candidate role.
Protect Candidate APIs.
Enforce backend authorization.
Protect private Candidate data.
Validate user inputs.
Secure uploaded resumes.
Prevent unauthorized application access.
21. Candidate Portal Acceptance Criteria

The Candidate Portal is complete when:

Candidate can access the Dashboard.
Candidate can manage their Profile.
Candidate can manage their Resume.
Candidate can search for Jobs.
Candidate can view Job Details.
Candidate can apply for Jobs.
Candidate can track Applications.
Candidate can view Interviews.
Candidate can manage Settings.
Candidate receives relevant notifications.
Candidate data is protected.
Candidate-only routes are secured.
Backend authorization is enforced.
The Portal is responsive.
The Portal is accessible.
