1. Purpose

The Candidate Settings page allows Candidates to manage their account, security, notifications, privacy, and account preferences.

2. Settings Access

Route:

/candidate/settings

Access requires:

Authenticated
+
Role = Candidate
+
Account Status = Active
3. Settings Structure
Candidate Settings
│
├── Account
├── Password & Security
├── Notifications
├── Privacy
└── Account Management
4. Account Settings

Candidates can manage basic account information.

Possible fields:

First Name
Last Name
Email
Phone Number

Actions:

Edit
Save Changes
Cancel

Sensitive information changes may require additional verification.

5. Password & Security

Candidates can change their password.

Fields:

Current Password
New Password
Confirm New Password

Flow:

Current Password
    ↓
New Password
    ↓
Confirm Password
    ↓
Validate
    ↓
Update Password

After a successful password change, the system should notify the Candidate.

6. Forgot Password

Candidates who cannot access their account can reset their password.

Login
    ↓
Forgot Password
    ↓
Enter Email
    ↓
Receive Reset Link
    ↓
Create New Password
    ↓
Login

Reset tokens must:

Be secure.
Expire.
Be single-use.
Be invalidated after successful use.
7. Notification Settings

Candidates can manage notification preferences.

Possible notification types:

Application Updates
Interview Invitations
Interview Reminders
Recruiter Messages
Job Recommendations
Platform Notifications

Example:

Application Updates     [ ON ]
Interview Notifications [ ON ]
Job Recommendations    [ OFF ]
Recruiter Messages     [ ON ]
8. Notification Channels

Where supported, Candidates may manage notification channels.

In-App Notifications
Email Notifications

The exact available channels should follow the Career77 notification system.

9. Privacy Settings

Candidates can manage their profile visibility.

Possible options:

Visible to Recruiters
Private

Example:

Profile Visibility

(•) Visible to Recruiters
( ) Private

The selected privacy setting must be respected throughout the platform.

10. Recruiter Discovery

If enabled, Candidates may control whether Recruiters can discover their profile.

Recruiter Discovery
[ ON ]

When disabled:

Candidate Profile
    ↓
Not available for Recruiter Discovery

The Candidate can still apply for jobs according to Career77 application rules.

11. Account Deactivation

Candidates may temporarily deactivate their account.

Flow:

Settings
    ↓
Deactivate Account
    ↓
Confirmation
    ↓
Account Deactivated

Confirmation message:

Are you sure you want to deactivate your account?

The system should clearly explain what happens after deactivation.

12. Account Deletion

If permanent account deletion is supported:

Settings
    ↓
Delete Account
    ↓
Confirmation
    ↓
Security Verification
    ↓
Account Deletion Process

The system should clearly explain:

What data will be deleted.
What data may be retained.
Whether deletion can be reversed.
How applications and records are handled.

The exact deletion policy must follow Career77 data retention requirements.

13. Logout

Candidates can securely log out.

Candidate Portal
    ↓
Logout
    ↓
Session Invalidated
    ↓
Redirect to Login

After logout, the previous session must not provide access to protected Candidate pages.

14. Settings Save Flow
Edit Setting
    ↓
Change Value
    ↓
Validate
    ↓
Save
    ↓
Success Message

Example:

Settings updated successfully.
15. Validation

Settings data must be validated before saving.

Validation may include:

Required fields.
Valid email format.
Valid phone number.
Password strength.
Matching password confirmation.
Valid preference values.

Invalid values should display clear error messages.

16. Loading State

While saving or loading settings:

Settings
    ↓
Loading
    ↓
Skeleton / Loading Indicator
    ↓
Settings Data

Save buttons should prevent accidental duplicate submissions.

17. Error State

If settings cannot be loaded:

Unable to load your settings.

Please try again.

Action:

[ Retry ]

If an update fails:

Unable to update your settings.

Please try again.

Internal system errors must not be exposed.

18. Security

The Settings feature must:

Require authentication.
Verify Candidate role.
Protect account information.
Require authorization for updates.
Secure password changes.
Protect password reset flows.
Prevent unauthorized account modifications.
Enforce backend authorization.

Sensitive actions may require additional authentication.

19. Responsive Design

Settings must support:

Desktop
Tablet
Mobile

Desktop:

Settings Navigation
        +
Settings Content

Mobile:

Settings Sections
        ↓
Selected Setting
        ↓
Settings Form

Forms must be easy to use on mobile devices.

20. Settings Acceptance Criteria

The Settings feature is complete when:

Candidate can manage account information.
Candidate can change their password.
Candidate can reset a forgotten password.
Candidate can manage notification preferences.
Candidate can manage privacy preferences.
Candidate can control recruiter discovery where supported.
Candidate can deactivate their account.
Account deletion follows the platform data policy.
Candidate can securely log out.
Settings changes are validated.
Success states are displayed.
Error states are handled.
Loading states are handled.
Sensitive actions are protected.
Backend authorization is enforced.
Settings are responsive.
Settings are accessible.

Candidate
