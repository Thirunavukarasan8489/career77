1. Purpose

The Candidate Dashboard is the main home page for authenticated Candidates.

It provides a quick overview of:

Profile completion
Job recommendations
Recent applications
Upcoming interviews
Notifications
Quick actions
2. Dashboard Access

The Dashboard requires:

Authenticated
+
Role = Candidate
+
Account Status = Active

Route:

/candidate/dashboard
3. Dashboard Layout
Candidate Dashboard
│
├── Welcome Section
│
├── Profile Completion
│
├── Quick Actions
│
├── Application Summary
│
├── Upcoming Interviews
│
├── Recommended Jobs
│
└── Recent Activity
4. Welcome Section

The Dashboard should welcome the Candidate.

Example:

Welcome back, Gokul 👋

Continue building your career and find your next opportunity.

The Candidate's first name may be displayed.

5. Profile Completion

Display the Candidate's profile completion percentage.

Example:

Profile Completion

████████░░ 80%

Your profile is almost complete.

The system may show missing items:

Add Skills
Add Experience
Upload Resume
Add Education

Each item should link to the relevant Profile section.

6. Quick Actions

The Dashboard should provide commonly used actions.

Search Jobs
Update Profile
Upload Resume
View Applications
View Interviews

Example:

[ Search Jobs ]

[ Update Profile ]

[ Upload Resume ]
7. Application Summary

Display a summary of the Candidate's applications.

Example:

Applications

Total          12
Under Review    5
Shortlisted     3
Interviews      2
Selected        1

Each summary item should link to the Applications page.

8. Recent Applications

Display the Candidate's latest applications.

Example:

Recent Applications

Frontend Developer
ABC Technologies
Applied 2 days ago
Status: Under Review

Backend Developer
XYZ Solutions
Applied 5 days ago
Status: Shortlisted

Each application should link to its application details.

9. Upcoming Interviews

Display upcoming interviews.

Example:

Upcoming Interviews

Full Stack Developer
ABC Technologies

Date: 25 July 2026
Time: 10:00 AM
Type: Video Interview

[ View Interview ]

If there are no upcoming interviews:

No upcoming interviews.
10. Recommended Jobs

Display relevant job opportunities.

Each job card may contain:

Job Title
Company
Location
Experience
Employment Type
Posted Date

Example:

Full Stack Developer
Career77 Technologies

Coimbatore
3–5 Years
Full Time

[ View Job ]

The Candidate can select a job to view the Job Detail page.

11. Recent Activity

Display important recent Candidate activities.

Examples:

Application submitted
Application status updated
Interview scheduled
Interview rescheduled
Resume updated
Profile updated

Example:

Today
Application status updated
Frontend Developer — Under Review

Yesterday
Interview scheduled
Backend Developer — 25 July
12. Notifications

The Dashboard may display recent notifications.

Examples:

New interview invitation
Application status updated
Recruiter message
Profile reminder

Unread notifications should be visually identifiable.

13. Empty States

The Dashboard must support empty states.

No Applications
You haven't applied for any jobs yet.

[ Search Jobs ]
No Interviews
You don't have any upcoming interviews.
No Recommended Jobs
No matching jobs available right now.

Try searching for more opportunities.
14. Dashboard Navigation

Recommended Candidate navigation:

Dashboard
Profile
Resume
Find Jobs
Applications
Interviews
Settings

The active navigation item should clearly indicate the current page.

15. Dashboard Loading State

While data is loading:

Dashboard
    ↓
Loading
    ↓
Skeleton UI
    ↓
Dashboard Data

The loading state should avoid unnecessary layout shifts.

16. Dashboard Error State

If dashboard data cannot be loaded:

Unable to load your dashboard.

Please try again.

Action:

[ Retry ]

The system must not expose internal API or database errors.

17. Dashboard Data

The Dashboard may retrieve:

Candidate Profile
Profile Completion
Application Summary
Recent Applications
Upcoming Interviews
Recommended Jobs
Recent Activity
Notifications

Only data belonging to the authenticated Candidate should be returned.

18. Dashboard Security

The Dashboard must:

Require authentication.
Verify Candidate role.
Verify account status.
Protect private Candidate information.
Enforce backend authorization.
Prevent access to another Candidate's data.

Example:

Candidate A
    ↓
Dashboard
    ↓
Candidate A Data Only
19. Responsive Design

The Dashboard must support:

Desktop
Tablet
Mobile

Desktop:

Sidebar
    +
Dashboard Content

Mobile:

Header
    ↓
Navigation
    ↓
Dashboard Cards
    ↓
Applications
    ↓
Interviews
    ↓
Recommended Jobs

Cards should stack vertically on smaller screens.

20. Dashboard Acceptance Criteria

The Candidate Dashboard is complete when:

Candidate can access the Dashboard.
Welcome message is displayed.
Profile completion is displayed.
Quick actions are available.
Application summary is displayed.
Recent applications are displayed.
Upcoming interviews are displayed.
Recommended jobs are displayed.
Recent activity is displayed.
Notifications are displayed where applicable.
Empty states are handled.
Loading states are handled.
Error states are handled.
Candidate data is private.
Backend authorization is enforced.
Dashboard is responsive.
Dashboard is accessible.
