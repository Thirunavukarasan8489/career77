6. Request Processing

Every protected request should follow:

Request
    ↓
Authenticate User
    ↓
Check Role / Permission
    ↓
Validate Input
    ↓
Execute Business Logic
    ↓
Database Operation
    ↓
Return Response
7. Authentication

Authentication should use:

NextAuth.js

The backend should identify the authenticated user before processing protected operations.

Authentication should support:

Candidate
Recruiter
Super Admin
8. Authorization

Authentication answers:

Who is the user?

Authorization answers:

What is the user allowed to do?

Authorization must be enforced on the server.

Do not rely only on frontend route protection.

9. Roles

Career77 has three primary roles:

Candidate
Recruiter
Super Admin

Each role has a different application experience and access level.

10. Candidate Backend

14. Business Logic Layer

Business logic should be separated from API request handling where practical.

Recommended flow:

    ↓
Validation
    ↓

    ↓

    ↓

Avoid placing complex business logic directly inside route handlers.

33. API Validation

All API inputs must be validated.

Validate:

Request Body
Query Parameters
Route Parameters
Uploaded Files

Validation should happen before business logic execution.

34. Error Handling

Backend errors should use consistent responses.

Example:

{
  "success": false,
  "message": "Unable to create job"
}

Avoid exposing internal implementation details to users.

49. Backend Architecture Principle

Career77 backend should follow:

Secure
    ↓
Validated
    ↓
Role-Aware
    ↓
Modular
    ↓
Scalable
    ↓
Maintainable

The backend should provide a reliable foundation for the three primary roles — Candidate, Recruiter, and Super Admin — while supporting jobs, applications, recruitment workflows, paid training, payments, subscriptions, notifications, and platform administration.
