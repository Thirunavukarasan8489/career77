2. Information Architecture Principles

The Career77 information architecture must follow these principles:

Keep the three primary authenticated roles clearly separated.
Keep public content accessible without authentication.
Keep authentication separate from authorization.
Use role-specific navigation after authentication.
Prevent unauthorized cross-role access.
Keep navigation consistent across related screens.
Use clear parent-child relationships between features.
Avoid unnecessary duplication of navigation items.
Keep feature ownership clear.
Ensure every core workflow can be completed through connected screens.
Maintain responsive navigation across mobile, tablet, and desktop.
Do not introduce additional top-level roles without explicit product approval.

18. Information Architecture Acceptance Criteria

The information architecture is considered complete when:

Public pages are clearly separated from authenticated portals.
Candidate navigation contains only approved Candidate features.
Recruiter navigation contains only approved Recruiter features.
Super Admin navigation contains only approved administrative features.
Authentication flows are separated by role.
Recruiter access uses the Request Access workflow.
Super Admin registration is not publicly available.
Candidate resources are protected.
Recruiter resources are restricted to authorized company resources.
Super Admin resources require secure administrative access.
Navigation is responsive.
Direct URL access is authorization-protected.
Cross-role workflows are connected.
Application data is connected to jobs, candidates, recruiters, companies, pipeline status, and interviews.
No additional top-level roles are introduced.
Navigation changes require explicit product approval.
