Career77 — Product Overview

Document Status: Client Approved
Document Type: Product Specification
Project: Career77
Product: Job Marketplace and Recruitment Platform
Current Scope: Production-ready Job Marketplace and Recruitment Platform
Primary Roles: Candidate, Recruiter, Super Admin
AI Features: Out of Current Scope

1. Purpose

Career77 is a modern job marketplace and recruitment platform designed to connect candidates with recruiters and companies through a unified digital hiring ecosystem.

The platform provides two primary business experiences:

Candidate Experience
Recruiter Experience

These experiences are supported by a third operational environment:

Super Admin Platform Management

Career77 must provide a consistent, responsive, secure, and scalable experience across:

Public / Marketing
Authentication
Candidate Portal
Recruiter Portal
Super Admin Console

The platform must be implemented as an integrated product rather than a collection of disconnected screens.

Every major feature must connect the appropriate:

Frontend
    ↓
API
    ↓
Backend Logic
    ↓
Database
    ↓
Authorization
    ↓
Related Role Workflow

A feature is considered complete only when the complete end-to-end workflow is functional.

2. Product Vision

The vision of Career77 is to create a reliable and easy-to-use recruitment ecosystem where:

Candidates can discover relevant job opportunities.
Candidates can build professional profiles.
Candidates can manage resumes.
Candidates can apply for jobs.
Candidates can track applications.
Candidates can manage interviews.
Recruiters can create and manage job openings.
Recruiters can review applicants.
Recruiters can manage hiring pipelines.
Recruiters can schedule interviews.
Companies can maintain professional public profiles.
Super Admins can manage and moderate the complete platform.
Recruiter organizations can use subscription and billing functionality.
The platform can scale to support future capabilities.

The current product must prioritize a strong core recruitment workflow before introducing advanced future capabilities.

6. Authentication Experience

Authentication supports the three approved primary roles:

Candidate
Recruiter
Super Admin

Authentication must remain separate from authorization.

The system must verify:

User identity.
User authentication status.
User role.
User permissions.
Resource access rights.

The authentication experience includes:

Authentication
│
├── Candidate
│   ├── Login
│   ├── Register
│   ├── Forgot Password
│   └── Reset Password
│
├── Recruiter
│   ├── Login
│   ├── Request Access
│   ├── Access Validation
│   └── Access Approval
│
└── Super Admin
    ├── Login
    ├── Forgot Password
    └── Secure Access

Authentication rules:

Candidate registration is self-service.
Recruiter registration is handled through Request Access.
Super Admin registration must never be publicly available.
Protected routes require valid authentication.
Authorization must be enforced on the frontend and backend.
Sensitive operations require appropriate permissions.
Backend APIs must never trust frontend authorization alone.

Standard analytics.

Future AI capabilities may be introduced as a separate product phase after explicit approval.

No AI dependencies, AI APIs, AI prompts, or AI UI components should be introduced into the current implementation without approval.
