1. Purpose

Candidate Authentication allows users to securely create and access a Candidate account.

The Candidate authentication flow supports:

Candidate Registration
Candidate Login
Email Verification
Forgot Password
Reset Password
Logout
Protected Candidate Access
2. Candidate Registration

Candidate registration collects basic account information.

Required Fields
First Name
Last Name
Email
Password
Confirm Password
Accept Terms & Conditions

    ↓

    ↓

    ↓
Create Account
    ↓

    ↓

    ↓
Candidate Dashboard

5. Email Verification

If email verification is enabled:

Registration
    ↓
Verification Email
    ↓
Candidate Opens Link
    ↓
Verify Email
    ↓
Account Activated

The verification link must:

Be secure.
Expire after a limited time.
Be usable only once.
6. Resend Verification Email

An unverified Candidate can request a new verification email.

Unverified Account
    ↓
Resend Verification
    ↓
Generate New Token
    ↓
Send Email

The request must be rate limited.

7. Forgot Password

The Candidate can reset a forgotten password.

Login
    ↓
Forgot Password
    ↓
Enter Email
    ↓
Submit
    ↓
Password Reset Email

The system should show a generic message:

If an account exists for this email,
password reset instructions will be sent.

The system should not reveal whether an email is registered.

8. Reset Password

The Candidate opens the password reset link.

Reset Link
    ↓
Validate Token
    ↓
Enter New Password
    ↓
Confirm Password
    ↓
Reset Password
    ↓
Login

The reset token must:

Be secure.
Expire.
Be single-use.
9. Candidate Password Rules

The password must follow the Career77 security policy.

Requirements:

Minimum password length.
Password confirmation.
Secure password hashing.
Never store plain-text passwords.
10. Candidate Logout

Candidate logout flow:

Candidate Dashboard
    ↓
Logout
    ↓
Invalidate Session
    ↓
Clear Authentication State
    ↓
Redirect to Public Page

After logout, protected Candidate pages must not remain accessible through the active session.

Authenticated
+

13. Candidate Authentication API

Suggested endpoints:

POST /api/auth/candidate/register
POST /api/auth/candidate/login
POST /api/auth/logout
GET  /api/auth/me
POST /api/auth/forgot-password
POST /api/auth/reset-password
POST /api/auth/verify-email
POST /api/auth/resend-verification

The final API structure must follow the main Career77 API specification.

14. Candidate Authentication Security

The system must:

Hash passwords securely.
Protect login endpoints with rate limiting.
Protect password reset endpoints.
Use secure authentication sessions.
Prevent account enumeration.
Validate all user inputs.
Protect protected routes.
Enforce authorization on the backend.
Never expose passwords or authentication secrets.
15. Candidate Job Application Authentication

A Candidate does not need to be logged in to browse jobs.

Public User
    ↓
Job Search
    ↓
Job Detail

Authentication is required when applying.

Job Detail
    ↓
Apply
    ↓
Is Candidate Logged In?
    │
    ├── Yes → Continue Application
    │
    └── No
          ↓
       Login / Register
          ↓
       Continue Application
16. Candidate Authentication Error States

The system should handle:

Invalid Email or Password
Email Not Verified
Account Suspended
Account Deactivated
Expired Verification Link
Expired Reset Link
Invalid Reset Link
Too Many Attempts
Server Error

Error messages must be clear but must not expose sensitive information.

17. Candidate Authentication Acceptance Criteria

Candidate Authentication is complete when:

Candidate can register.
Candidate can log in.
Candidate can log out.
Candidate can verify email.
Candidate can resend verification email.
Candidate can reset password.
Candidate can recover a forgotten password.
Candidate account status is enforced.
Candidate-only routes are protected.
Backend authorization is enforced.
Authentication sessions are secure.
Login attempts are rate limited.
Passwords are securely hashed.
Authentication errors are handled.
Job application requires Candidate authentication.
Candidate is redirected to the correct dashboard after login.
