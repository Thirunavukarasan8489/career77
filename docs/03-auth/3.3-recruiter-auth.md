1. Purpose

Recruiter Authentication manages the registration, company setup, verification, approval, and login process for Recruiters.

Unlike Candidates, Recruiters must be verified by the Career77 Super Admin before accessing the Recruiter Portal.

3. Recruiter Registration

Recruiters can create an account through the Recruiter Registration page.

Route:

/recruiter/register

Registration flow:

Recruiter
    ↓
Register
    ↓
Enter Details
    ↓
Validate Details
    ↓
Create Account

Public Recruiter registration does not immediately provide access to the Recruiter Portal.

4. Recruiter Details

The Recruiter should provide basic information.

Possible fields:

First Name
Last Name
Work Email
Phone Number
Password
Confirm Password

The system should validate all required information before account creation.

5. Company Setup

During registration, the Recruiter must provide company information.

Possible fields:

Company Name
Company Email
Company Phone
Company Website
Industry
Company Size
Company Location
Company Description
Company Logo

The Company Profile may require additional verification documents depending on Career77 verification requirements.

6. Account Creation

After successful validation:

Recruiter Details
        +
Company Details
        ↓
Validation
        ↓
Create Recruiter Account
        ↓
Create Company Profile
        ↓
Verification Request Created

The Recruiter account and Company Profile should be linked.

Recruiter
    ↓
Company

A Recruiter should not receive full Recruiter Portal access until Super Admin verification is completed.

7. Verification Status

The Recruiter verification request may have the following statuses:

Pending
Under Review
Approved
Rejected
Resubmitted

Initial status:

Account Created
    ↓
Verification Status = Pending
8. Super Admin Verification

The Super Admin reviews the Recruiter and Company information.

The Super Admin may review:

Recruiter Details
Company Details
Company Website
Company Information
Verification Documents

The Super Admin can:

Approve
Reject
Request More Information
9. Verification Approval

When the Super Admin approves the Recruiter:

Super Admin
    ↓
Approve Recruiter
    ↓
Verification Status = Approved
    ↓
Recruiter Account Activated
    ↓
Recruiter Portal Access Enabled
    ↓
Welcome Email Sent
    ↓
Recruiter Login

The welcome email should contain:

Welcome Message
Career77 Recruiter Portal
Login Information
Login Button / Link
Basic Next Steps

The system should not send passwords through email.

10. Recruiter Login

After approval, the Recruiter can log in.

Route:

/recruiter/login

Login fields:

Work Email
Password

Flow:

Recruiter
    ↓
Login
    ↓
Validate Credentials
    ↓
Check Account Status
    ↓
Check Verification Status
    ↓
Create Secure Session
    ↓
Recruiter Dashboard
11. Verification Check During Login

The system must check the Recruiter's verification status.

Login
    ↓
Credentials Valid?
    │
    ├── No → Invalid Credentials
    │
    └── Yes
          ↓
    Verification Status?
          │
          ├── Pending → Verification Pending
          │
          ├── Under Review → Under Review
          │
          ├── Rejected → Request Review / Resubmit
          │
          └── Approved → Recruiter Dashboard

Only approved Recruiters can access the Recruiter Portal.

12. Rejected Verification

If the Super Admin rejects the Recruiter's verification request:

Super Admin
    ↓
Reject
    ↓
Add Rejection Reason
    ↓
Recruiter Notified

The Recruiter should be able to view the rejection status.

Example:

Verification Status: Rejected

Reason:
Additional company verification information is required.
13. Rejection Request / Re-Submission

A rejected Recruiter can request another review.

Flow:

Rejected
    ↓
View Rejection Reason
    ↓
Update Required Information
    ↓
Submit Review Request
    ↓
Verification Status = Resubmitted
    ↓
Super Admin Review

The Recruiter may update the information requested by the Super Admin before resubmitting.

14. Request Review

The Recruiter may submit a review request after rejection.

Example:

Verification Rejected

[ Update Information ]

[ Request Review ]

The request should include:

Request Message
Updated Company Information
Updated Documents

The Super Admin can review the updated request.

15. Re-Approval

If the Super Admin approves the resubmitted request:

Resubmitted
    ↓
Super Admin Review
    ↓
Approved
    ↓
Recruiter Account Activated
    ↓
Welcome Email Sent
    ↓
Recruiter Login
    ↓
Recruiter Dashboard
16. Pending Verification

If the Recruiter has not yet been reviewed:

Verification Status: Pending

Your account has been created successfully.
Your company verification request is waiting for
Super Admin review.

The Recruiter should not access protected Recruiter Portal features during this stage.

17. Under Review

If the Super Admin is currently reviewing the account:

Verification Status: Under Review

Your company verification is currently being reviewed.

The Recruiter should wait for the Super Admin decision.

18. Recruiter Portal Access

Access requirement:

Authenticated
+
Role = Recruiter
+
Account Status = Active
+
Verification Status = Approved

If any required condition fails, access must be denied.

Recruiter Portal
    ↓
Authentication Check
    ↓
Role Check
    ↓
Account Status Check
    ↓
Verification Status Check
    ↓
Access Granted
19. Recruiter Dashboard Redirect

After successful login:

Recruiter Login
    ↓
Credentials Valid
    ↓
Account Approved
    ↓
Account Active
    ↓
/recruiter/dashboard

The Recruiter is redirected to the Recruiter Dashboard.

21. Recruiter Logout

The Recruiter can securely log out.

Recruiter Dashboard
    ↓
Logout
    ↓
Invalidate Session
    ↓
Clear Authentication State
    ↓
Redirect to Recruiter Login

After logout, protected Recruiter pages must not remain accessible through the previous session.

22. Forgot Password

Recruiters can reset forgotten passwords.

Recruiter Login
    ↓
Forgot Password
    ↓
Enter Work Email
    ↓
Password Reset Email
    ↓
Reset Password
    ↓
Recruiter Login

Reset tokens must:

Be secure.
Expire.
Be single-use.
Be invalidated after successful reset.
23. Recruiter Authentication Security

The system must:

Validate Recruiter registration data.
Validate Company information.
Prevent duplicate accounts where applicable.
Securely hash passwords.
Protect password reset.
Rate limit login attempts.
Protect verification data.
Verify Recruiter role.
Verify account status.
Verify approval status.
Enforce backend authorization.
Protect Recruiter Portal routes.
Record important authentication and verification events.
24. Recruiter Authentication Acceptance Criteria

The Recruiter Authentication system is complete when:

Recruiter can register.
Recruiter can enter personal details.
Recruiter can enter Company details.
Recruiter account is created successfully.
Company Profile is created and linked.
Verification request is created.
Super Admin can review the request.
Super Admin can approve the Recruiter.
Super Admin can reject the Recruiter.
Rejection reason can be provided.
Recruiter can request review after rejection.
Recruiter can resubmit required information.
Approved Recruiter receives a welcome email.
Approved Recruiter can log in.
Pending Recruiter cannot access the Recruiter Portal.
Rejected Recruiter cannot access the Recruiter Portal.
Only approved Recruiters can access the Recruiter Portal.
Recruiter is redirected to the Dashboard after successful login.
Recruiter can reset their password.
Recruiter can securely log out.
Backend authorization is enforced.
Authentication and verification events are logged.
