1. Purpose

This document defines responsive design standards for Career77.

Career77 must provide a consistent experience across:

Desktop
Laptop
Tablet
Mobile
responsive for landscape and portrait mode in all device . eg. mac , windows, android ,ios (mobile and tablet ) should be checked and verified. 

Responsive behavior applies to:

Public pages
Candidate portal
Recruiter portal
Super Admin portal
Authentication
Forms
Tables
Cards
Navigation
Modals
Filters
2. Responsive Principles

Career77 responsive design should follow:

Mobile-first development
Flexible layouts
Consistent spacing
Readable content
Touch-friendly controls
Simple navigation
No unnecessary horizontal scrolling
3. Mobile-First Approach

Components should be designed for smaller screens first.

Mobile
    ↓
Tablet
    ↓
Desktop

Use responsive breakpoints to progressively enhance the layout.

4. Responsive Breakpoints

Recommended breakpoints:

Mobile
< 640px

Tablet
640px – 1023px

Desktop
1024px – 1279px

Large Desktop
1280px+

The exact breakpoint should depend on content requirements.

5. Mobile Layout

Mobile screens should use:

Single Column
    ↓
Full Width Content
    ↓
Compact Navigation
    ↓
Stacked Actions

Avoid complex multi-column layouts on small screens.

6. Tablet Layout

Tablet layouts may use:

Two Column Layout
    ↓
Condensed Sidebar
    ↓
Responsive Grid

Components should adapt based on available width.

7. Desktop Layout

Desktop layouts can use:

Sidebar
+
Main Content

or:

Filter Panel
+
Content Area

Use available screen space effectively without creating excessive content width.

8. Large Desktop

Large screens should not simply stretch content indefinitely.

Use:

Maximum Container Width
    +
Centered Content

Maintain comfortable reading widths.

9. Responsive Container

Recommended horizontal padding:

Mobile
16px

Tablet
24px

Desktop
32px

Containers should use a maximum width where appropriate.

10. Responsive Grid

Recommended grid structure:

Mobile
1 Column

Tablet
2 Columns

Desktop
3–4 Columns

Grid layouts should automatically adapt to available space.

Desktop:

Expanded Sidebar
    +
Main Content

Tablet:

13. Mobile Navigation

Mobile navigation may use:

Menu button
Drawer
Bottom navigation

Candidate portal may prioritize:

Home
Jobs
Applications

Other options can be accessed through the menu.

14. Recruiter Mobile Navigation

Recruiter mobile navigation may prioritize:

Dashboard
Jobs
Applicants
Pipeline
Messages

Additional items should be available through the navigation drawer.

15. Admin Mobile Navigation

Admin interfaces are data-heavy.

On mobile:

Header
    ↓
Menu
    ↓
Page Content

Complex admin navigation should use a drawer.

16. Responsive Header

Desktop header:

Logo | Navigation | Search | Actions | Profile

Mobile header:

Menu | Logo | Profile

Navigation links should move into a mobile menu.

17. Responsive Search

Desktop job search:

[Keyword] [Location] [Experience] [Search]

Mobile:

[Search Jobs]

[Filters]

Advanced filters should open in a drawer or modal.

21. Responsive Multi-Step Forms

Multi-step forms should simplify on mobile.

Desktop:

1 ─── 2 ─── 3 ─── 4 ─── 5

Mobile:

Step 2 of 5

Job Details

Use a compact progress indicator on smaller screens.

22. Responsive Tables

Tables should not force unnecessary horizontal scrolling.

Possible strategies:

Desktop
Full Table

Tablet
Reduced Columns

Mobile
Card/List View

For complex data, horizontal scrolling may be used when necessary.

23. Responsive Applicant Table

Desktop:

Candidate | Match | Experience | Status | Actions

Mobile:

Candidate
Match Score
Experience
Status
[View]

Important information should appear first.

24. Responsive Hiring Pipeline

Desktop:

Applied | Screening | Interview | Offer | Hired

Mobile options:

Stage Selector
    ↓
Candidate List

or:

Horizontal Scrollable Pipeline

The selected approach should preserve usability.

25. Responsive Cards

Cards should adapt their internal layout.

Desktop:

Horizontal

Mobile:

Vertical

Avoid shrinking text too much to maintain a horizontal layout.

26. Responsive Buttons

Desktop:

[Save] [Cancel]

Mobile:

[Save]

[Cancel]

Buttons may become full width when appropriate.

27. Touch Targets

Interactive elements should provide comfortable touch targets.

Use sufficiently large:

Buttons
Links
Icons
Checkboxes
Navigation items

Avoid placing interactive controls too close together.

28. Responsive Typography

Typography should scale based on screen size.

Example:

Desktop
Large Heading

Tablet
Medium Heading

Mobile
Compact Heading

Do not reduce text size so much that readability suffers.

37. Responsive Images

Images should:

Maintain aspect ratio
Scale within containers
Avoid distortion
Use responsive sizes

Company logos and profile images should have consistent aspect ratios.

38. Responsive Media

Videos and embedded content should use responsive containers.

Avoid fixed dimensions that cause overflow.

39. Responsive Spacing

Recommended:

Mobile
Compact Spacing

Tablet
Medium Spacing

Desktop
Large Spacing

Use the spacing tokens defined in design-spacing.md.

40. Responsive Overflow

Avoid unexpected horizontal overflow.

Check:

Long text
Tables
Buttons
Navigation
Cards
Images
Modals
Forms

Use wrapping, stacking, truncation, or scrolling where appropriate.

41. Long Text

Long text should be handled using:

Wrapping
Truncation
Ellipsis
Expandable Content

Never allow long content to break the layout.

42. Responsive Accessibility

Responsive layouts must preserve:

Keyboard navigation
Focus visibility
Reading order
Screen reader access
Touch accessibility

Changing layout should not change the logical content order.

43. Responsive Testing

Test major screens at minimum on:

Mobile
Small Phone

Tablet
Portrait / Landscape

Desktop
Laptop

Large Desktop
Wide Monitor

Test both portrait and landscape where applicable.

44. Responsive Testing Checklist

Verify:

✓ No Unexpected Horizontal Scroll
✓ Navigation Works
✓ Buttons Are Accessible
✓ Forms Are Usable
✓ Text Is Readable
✓ Images Scale Correctly
✓ Tables Remain Usable
✓ Modals Fit the Screen
✓ Filters Are Accessible
✓ Touch Targets Are Comfortable
45. Performance

Responsive pages should avoid loading unnecessary resources.

Use:

Responsive images
Lazy loading
Dynamic imports where appropriate
Optimized assets

Mobile users should receive an efficient experience.

46. Next.js Responsive Implementation

Use responsive Tailwind utilities.

Example:

grid-cols-1
md:grid-cols-2
lg:grid-cols-3

Example:

p-4
md:p-6
lg:p-8

Responsive behavior should be defined at the component level where possible.

47. Future Dark Theme

The current Career77 design supports Light Theme only.

Responsive behavior should remain independent of the theme.

When Dark Theme is introduced:

Responsive breakpoints remain unchanged
Layout structure remains unchanged
Mobile behavior remains unchanged
Component responsiveness remains unchanged

Only theme-related visual tokens should change.

48. Final Responsive Principle

Career77 should follow:

Mobile First
    ↓
Flexible Layout
    ↓
Responsive Components
    ↓
Touch Friendly
    ↓
Accessible
    ↓
Desktop Enhanced

The goal is to provide a consistent and usable Career77 experience across all screen sizes while adapting navigation, content density, forms, tables, filters, and actions to the available space.
