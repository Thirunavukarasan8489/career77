1. Purpose

This document defines typography standards for the Career77 platform.

Typography should provide:

Clear hierarchy
Easy readability
Strong visual identity
Fast content scanning
Consistent UI
Professional enterprise experience

2. Typography Direction

Career77 uses a modern sans-serif typography style inspired by:

Inter
Outfit

The final production font should be selected consistently across the platform.

Use one primary font family throughout the application unless a specific design requirement exists.

3. Font Family

Recommended primary font:

Inter

Alternative:

Outfit

Typography should remain consistent across:

Public Pages
Candidate Portal
Recruiter Portal
Super Admin Portal
Authentication
Emails

5. Display Text

Used for major public-facing hero sections.

Example:

Find Your Next Career Opportunity

Recommended characteristics:

Large Size
Bold Weight
Tight Line Height
Strong Contrast

Use primarily for:

Landing page hero
Major marketing sections
Campaign pages

Examples:

Recommended:

Line Height: Tight

Used for major sections.

Examples:

Recommended Jobs

Recommended:

Font Weight: Bold / Semibold
Line Height: Tight
8. Heading 3

Used for subsections.

Examples:

Personal Information

Recommended:

Line Height: Normal

Used for smaller content groups.

Examples:

Skills

10. Body Text

Body text should prioritize readability.

Use for:

Descriptions
Job Details
Company Information
Instructions
Dashboard Content

Recommended:

Font Weight: Regular
Line Height: Relaxed

Avoid overly small body text.

Recommended:

1.0–1.1

1.1–1.25

16. Letter Spacing

Use letter spacing carefully.

For large headings:

Tight

For body text:

Normal

For uppercase labels:

Slightly Increased

Avoid excessive letter spacing.

17. Text Color

Primary text:

Dark Slate
#0F172A

Secondary text should use a softer neutral.

Examples:

Muted Text
Supporting Text
Metadata

Text contrast must remain accessible.

18. Link Typography

Links should be visually identifiable.

Use:

Primary Brand Color

Example:

View Job
View Profile
Forgot Password?
Learn More

Links should include hover and focus states.

19. Button Typography

Medium / Semibold

Examples:

Apply Now
Create Job
Publish Job
Save Changes
Continue

Button text should be short and action-oriented.

25. Recruiter Table Typography

Enterprise tables should prioritize density and readability.

Use:

Table Header
Medium / Semibold

Table Content
Regular / Medium

Status
Medium

Actions
Medium

Avoid excessively large typography in data-heavy tables.

26. Status Typography

Status labels should be short.

Examples:

Active
Pending
Approved
Rejected
Draft
Closed
Expired

Use typography with status indicators.

Example:

● Approved
● Pending
● Rejected

Do not rely only on color to communicate status.

27. Error Typography

Error messages should be:

Clear
Short
Readable
Actionable

Example:

Please enter a valid email address.

Avoid technical error messages for end users.

28. Success Typography

Success messages should clearly communicate completion.

Examples:

Job published successfully.
Application submitted successfully.
Payment completed successfully.

Keep success messages concise.

29. Empty State Typography

Empty states should contain:

Title
Supporting Message
Optional Action

Example:

No Applications Yet

Your job applications will appear here.

[Explore Jobs]
30. Responsive Typography

Typography should adapt across screen sizes.

Desktop:

Larger Headings
More Horizontal Space

Mobile:

Smaller Headings
Shorter Line Length
Improved Readability

Avoid large hero headings that overflow on mobile screens.

31. Mobile Typography

Recommended mobile adjustments:

Display
36–48px

Heading 1
30–36px

Heading 2
24–30px

Heading 3
20–24px

Body
16px

Small Text
14px

Exact values should follow the screen design.

32. Text Width

Long text should not span the entire screen.

For readable content:

Maximum Line Length
Approximately 60–80 Characters

Use constrained content widths for:

Articles
Job Descriptions
Company Descriptions
Help Content
CMS Content
33. Text Alignment

Use:

Left Alignment

as the default for most content.

Center alignment may be used for:

Hero Sections
Empty States
Authentication Screens
Marketing Sections

Avoid justified text unless specifically required.

34. Text Transformation

Avoid unnecessary uppercase text.

Use uppercase selectively for:

Small Labels
Categories
Section Metadata

Do not use all-uppercase text for large paragraphs.

35. Truncation

Long content should be handled carefully.

For cards and tables:

Text Truncation
Line Clamp
Tooltip
View More

Example:

Senior Full Stack Developer with experience...

Users should have a way to access the complete content.

36. Typography Accessibility

Typography must support accessibility.

Ensure:

Sufficient contrast
Readable font sizes
Clear hierarchy
Adequate line height
Visible focus states
No information communicated only through font styling
37. Font Loading

Fonts should be loaded efficiently.

Avoid unnecessary font families and weights.

Prefer loading only required weights.

Example:

Regular
Medium
Semibold
Bold

This helps reduce page load size.

38. Tailwind Typography Mapping

Typography tokens should map consistently to Tailwind utilities.

Example:

Display
text-5xl / text-6xl

Heading
text-3xl / text-4xl

Body
text-base

Small
text-sm

Caption
text-xs

Use responsive classes where appropriate.

39. Typography Consistency

Developers should avoid arbitrary typography values.

Before adding a new text size:

    ↓
Create New Token Only if Required
