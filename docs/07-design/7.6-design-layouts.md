1. Purpose

This document defines layout standards for the Career77 platform.

The layout system provides consistent structure across:

Public pages
Candidate portal
Recruiter portal
Super Admin portal
Authentication
Job search
Job details
Training
Billing
Support
2. Layout Principles

Career77 layouts should be:

Clean
Structured
Responsive
Easy to scan
Consistent
Accessible

The layout should prioritize the user's primary task.

42. Layout Hierarchy

Every page should clearly communicate:

Where am I?
    ↓
What information am I viewing?
    ↓
What action can I take?
    ↓
What happens next?
43. Layout Consistency

Common layouts should be reused.

Example:

PublicLayout
CandidateLayout
RecruiterLayout
AdminLayout
AuthLayout

Avoid creating different layout structures for every individual page without a clear reason.

44. Layout Component Structure

Recommended:

app/
├── (public)/
├── (auth)/
├── (candidate)/
├── (recruiter)/
└── (admin)/

Each route group may use its own layout while sharing global design components.

45. Layout and Accessibility

Layouts must support:

Keyboard navigation
Logical tab order
Screen readers
Visible focus
Responsive navigation
Accessible landmarks

Use semantic structures such as:

header
nav
main
aside
section
footer

where appropriate.

46. Layout and Future Dark Theme

The current design system supports Light Theme only.

Layouts should remain theme-independent.

When Dark Theme is introduced later:

Layout structure should remain unchanged
Spacing should remain unchanged
Grid structure should remain unchanged
Component positioning should remain unchanged

Only visual theme tokens should be updated where possible.

47. Final Layout Principle

Career77 layouts should follow:

Public
    ↓
Discover and Convert

Candidate
    ↓
Discover, Apply, Track

Recruiter
    ↓
Find, Evaluate, Hire

Super Admin
    ↓
Manage, Verify, Control

Every layout should be designed around the user's primary goal while maintaining a consistent Career77 visual system across the entire platform.
