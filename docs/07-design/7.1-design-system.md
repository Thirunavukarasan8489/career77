1. Purpose

This document defines the visual and interaction foundation for the Career77 platform.

The design system ensures a consistent experience across:

Public pages
Candidate portal
Recruiter portal
Super Admin portal
Authentication
Job search
Job applications
Hiring workflows
Paid training
Billing
Support
2. Design Direction

Career77 follows an:

Elite Tech-Modern
Minimalist Premium
High Conversion
Enterprise SaaS

design direction.

The interface should feel:

Professional
Modern
Trustworthy
Fast
Clean
Intelligent
Easy to navigate
3. Core Design Principles
3.1 Clarity

Users should immediately understand:

Where they are
What they can do
What action to take next
3.2 Simplicity

Avoid unnecessary visual elements.

Prefer:

Clear Layout
+
Strong Hierarchy
+
Focused Actions
3.3 Consistency

Use shared:

Colors
Typography
Spacing
Components
Icons
Interaction patterns
3.4 Accessibility

Design should be usable by as many people as possible.

Consider:

Keyboard navigation
Color contrast
Focus states
Screen readers
Readable typography
3.5 Responsive Design

The interface must work across:

Mobile
Tablet
Desktop
Large Desktop
3.6 Conversion Focus

Public Career77 pages should guide users toward meaningful actions.

Examples:

Search Jobs
View Job
Apply
Create Candidate Account
Register as Recruiter
Explore Training
4. Design Tokens

Career77 should use centralized design tokens.

Color Tokens
Typography Tokens
Spacing Tokens
Radius Tokens
Shadow Tokens
Border Tokens
Breakpoint Tokens

Tokens should be reused instead of defining random values throughout the application.

7. Primary Actions

Primary actions should use the primary brand accent.

Examples:

Apply Now
Create Job
Publish Job
Save Changes
Continue
Start Training

Primary actions should be:

Clearly visible
Easy to identify
Consistent across the application
8. Secondary Actions

Secondary actions should not compete with the primary action.

Examples:

Cancel
Back
View Details
Edit
Save Draft

Use neutral or outlined styles where appropriate.

9. Destructive Actions

Destructive actions must clearly communicate risk.

Examples:

Delete Job
Reject Recruiter
Remove User
Cancel Subscription
Delete Account

Use confirmation dialogs for irreversible actions.

10. Status Colors

Status colors should be consistent.

Recommended semantic categories:

Success
Warning
Error
Info
Neutral

Do not rely on color alone.

Use:

Color
+
Icon
+
Text

for important status information.

11. Cards

Cards should be used to group related information.

Examples:

Job Card
Candidate Card
Company Card
Dashboard Metric Card
Training Card
Application Card

Cards should have:

Clear hierarchy
Consistent padding
Appropriate border
Subtle shadow where required
Consistent radius

Avoid excessive card nesting.

12. Forms

Forms should be simple and easy to complete.

Each form should include:

Label
Input
Helper Text Where Required
Validation
Error Message

Important forms:

Candidate Registration
Recruiter Registration
Job Creation
Candidate Profile
Company Profile
Payment
Training Enrollment

Examples:

Users
Companies

Tables should support:

Clear headers
Sorting where needed
Filtering where needed
Pagination
Row actions
Responsive behavior

Navigation should clearly indicate the current location.

Use:

Active State
Hover State
Focus State
Selected State

Candidate, Recruiter, and Admin portals may use different navigation structures while following the same design system.

16. AI Experience

Career77 is an AI-assisted job platform.

AI recommendations should feel:

Helpful
Contextual
Explainable
Non-intrusive

Examples:

AI Job Match
AI Candidate Match
Resume Suggestions
Profile Improvement
Skill Recommendations
Hiring Insights

AI elements should support the user rather than overwhelm the main workflow.

17. Empty States

Every major data-driven page should have an empty state.

Examples:

No Applications
No Jobs
No Applicants
No Interviews
No Notifications
No Training Enrollments

Empty states should explain:

What happened
+
What the User can do next
18. Loading States

Use appropriate loading indicators.

Examples:

Skeleton
Spinner
Progress Indicator
Button Loading State

Avoid unnecessary full-page loading screens.

19. Error States

Error states should:

Explain the problem
Avoid technical details
Provide a recovery action

Example:

Unable to load applications.

Please try again.
20. Microinteractions

Examples:

Button Hover
Card Hover
Focus States
Smooth Transitions

Modal Animation
Loading Feedback

Avoid excessive animation.

21. Motion

Use animation to communicate:

State Change
Navigation
Feedback
Loading
Success

Avoid animations that distract from important tasks.
