1. Purpose

The Hiring Pipeline allows Recruiters to manage Candidates through different stages of the recruitment process.

Recruiters can:

View Candidates by Stage
Move Candidates Between Stages
Review Candidate Information
Add Internal Notes
Schedule Interviews
Track Candidate Progress
Manage Hiring Decisions
2. Hiring Pipeline Access

Route:

/recruiter/hiring-pipeline

Access requires:

Authenticated
+
Role = Recruiter
+
Account Status = Active
+
Verification Status = Approved

8. Move Candidate

Recruiters can move Candidates between stages.

Example:

Screening
    ↓
Shortlisted

The Recruiter may move the Candidate using:

Drag and Drop

or:

Move Candidate
    ↓
Select New Stage
    ↓
Confirm

Every stage change should be recorded.

9. Pipeline Stage Rules

Each stage should have clear meaning.

Applied

Candidate has submitted an Application.

Screening

Recruiter is reviewing the Candidate.

Shortlisted

Candidate has been selected for further evaluation.

Interview

Candidate is participating in the Interview process.

Assessment

Candidate is completing a required Assessment.

Offer

Recruiter has moved the Candidate to the Offer stage.

Hired

Candidate has successfully completed the hiring process.

Rejected

Candidate is no longer being considered.

10. Candidate Withdrawal

Candidates may withdraw their Applications.

Candidate
    ↓
Withdraw Application
    ↓
Application Status = Withdrawn

The Recruiter should see the updated status.

The Recruiter should not be able to move a withdrawn Candidate through the active hiring process unless the Candidate submits a new Application or Career77 explicitly supports reopening.

11. Candidate Details

Recruiters can open a Candidate from the Pipeline.

The Candidate Details view may include:

Candidate Profile
Resume
Application
Skills
Experience
Education
Interview History
Internal Notes
Activity History
12. Internal Notes

Recruiters can add private notes about Candidates.

Example:

Internal Notes

Strong technical skills.
Good experience with React and Node.js.
Proceed to technical interview.

Internal notes are only visible to authorized Recruiters and Company team members.

Candidates must not be able to access internal notes.

13. Activity Timeline

The Pipeline should maintain an activity history.

Example:

23 July
Moved to Screening

24 July
Candidate shortlisted

25 July
Interview scheduled

26 July
Moved to Interview

This provides Recruiters with a complete Candidate journey.

14. Interview Integration

Recruiters can schedule Interviews directly from the Pipeline.

Candidate
    ↓
Shortlisted
    ↓
Schedule Interview
    ↓
Interview Scheduled
    ↓
Move to Interview

Interview information should be connected to the Candidate's Application.

15. Assessment Integration

If an Assessment is required:

Interview
    ↓
Assessment
    ↓
Candidate Completes Assessment
    ↓
Recruiter Reviews Result
    ↓
Move to Offer / Reject

Assessment results should be visible only to authorized Recruiters.

16. Offer Stage

When a Candidate reaches the Offer stage:

Offer
    ↓
Prepare Offer
    ↓
Send Offer
    ↓
Candidate Decision

Possible outcomes:

Accepted
Rejected
Expired

If accepted:

Offer Accepted
    ↓
Hired

If rejected:

Offer Rejected
    ↓
Rejected
17. Hired Stage

When the Candidate is successfully hired:

Candidate
    ↓
Offer Accepted
    ↓
Hired

The Candidate's Application status should be updated accordingly.

The hiring activity should be recorded in the system.

18. Rejection

Recruiters can reject Candidates from appropriate Pipeline stages.

Candidate
    ↓
Reject
    ↓
Confirmation
    ↓
Status = Rejected

The Recruiter may provide an internal rejection reason.

The Candidate may receive a notification based on Career77's notification policy.

23. Empty State

If the selected Job has no Candidates:

No Candidates in the Hiring Pipeline.

Candidates will appear here when they apply
for this Job.

Action:

24. Loading State

While Pipeline data is loading:

Hiring Pipeline
    ↓
Loading
    ↓
Skeleton Pipeline Columns
    ↓
Pipeline Loaded

The interface should remain stable during loading.

25. Error State

If Pipeline data cannot be loaded:

Unable to load the hiring pipeline.

Please try again.

Action:

[ Retry ]

Internal system errors must not be exposed.

26. Pipeline Data Security

Recruiters must only access Candidates associated with Jobs belonging to Companies they are authorized to manage.

Recruiter
    ↓
Company Authorization
    ↓
Job Authorization
    ↓
Candidate Application
    ↓
Pipeline Access

Recruiters must not access another Company's Pipeline.

Backend authorization is mandatory.

29. Hiring Pipeline Acceptance Criteria

The Hiring Pipeline is complete when:

Recruiters can select a Job.
Recruiters can view Candidates by stage.
Recruiters can move Candidates between stages.
Drag-and-drop is supported where implemented.
Candidate details can be viewed.
Resumes can be accessed.
Internal notes can be added.
Activity history is displayed.
Interviews can be scheduled.
Assessments can be managed where implemented.
Offers can be managed where implemented.
Candidates can be marked as Hired.
Candidates can be Rejected.
Candidate Withdrawals are handled.
Pipeline search is available.
Pipeline filters are available.
Pipeline sorting is available.
Audit Logs are recorded.
Candidate privacy is respected.
Company data access is protected.
Empty states are handled.
Loading states are handled.
Error states are handled.
Backend authorization is enforced.
Pipeline is responsive.
Pipeline is accessible.
