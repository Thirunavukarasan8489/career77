1. Purpose

The Recruiter Dashboard is the main workspace for Recruiters after successful login.

It provides a quick overview of:

Active Jobs
Applications
New Applicants
Shortlisted Candidates
Upcoming Interviews
Hiring Pipeline
Recruitment Analytics
Recent Activity
2. Dashboard Access

Route:

/recruiter/dashboard

Access requires:

Authenticated
+
Role = Recruiter
+
Account Status = Active
+
Verification Status = Approved
3. Dashboard Layout
Recruiter Dashboard
│
├── Welcome Section
├── Quick Actions
├── Recruitment Overview
├── Active Jobs
├── Application Summary
├── Hiring Pipeline
├── Upcoming Interviews
├── Recommended Candidates
└── Recent Activity
4. Welcome Section

The Dashboard should display a personalized welcome message.

Example:

Welcome back, Gokul 👋

Manage your hiring process and find the right talent for your team.

The Recruiter's name and Company name may be displayed.

5. Quick Actions

The Dashboard should provide quick access to common actions.

Create Job
View Applicants
Open Hiring Pipeline
Schedule Interview
Manage Company
View Analytics

Example:

[ Create Job ]

[ View Applicants ]

[ Hiring Pipeline ]

17. Loading State

While Dashboard data is loading:

Recruiter Dashboard
    ↓
Loading
    ↓
Skeleton UI
    ↓
Dashboard Data

The Dashboard should avoid unnecessary layout shifts.

18. Error State

If Dashboard data cannot be loaded:

Unable to load your dashboard.

Please try again.

Action:

[ Retry ]

Internal API, database, or server errors must not be exposed to Recruiters.

19. Dashboard Data

The Dashboard may retrieve:

Recruiter Information
Company Information
Active Jobs
Application Summary
Hiring Pipeline
Upcoming Interviews
Recommended Candidates
Recent Activity
Notifications

Only data belonging to the Recruiter's authorized Company should be displayed.

20. Data Authorization

Recruiters must only access Dashboard data they are authorized to view.

Example:

Recruiter A
    ↓
Company A
    ↓
Company A Dashboard Data

Recruiter A must not access:

Company B
Company B Jobs
Company B Applicants
Company B Analytics

Backend authorization is mandatory.

22. Dashboard Security

The Dashboard must:

Require authentication.
Verify Recruiter role.
Verify account status.
Verify Super Admin approval.
Protect Company information.
Protect Candidate information.
Enforce backend authorization.
Prevent unauthorized Company data access.
23. Dashboard Acceptance Criteria

The Recruiter Dashboard is complete when:

Approved Recruiter can access the Dashboard.
Welcome section is displayed.
Quick Actions are available.
Recruitment metrics are displayed.
Active Jobs are displayed.
Application Summary is displayed.
Hiring Pipeline overview is displayed.
Upcoming Interviews are displayed.
Recommended Candidates are displayed where available.
Recent Activity is displayed.
Notifications are displayed.
Job performance information is displayed where implemented.
Empty states are handled.
Loading states are handled.
Error states are handled.
Company data access is restricted.
Candidate data is protected.
Backend authorization is enforced.
Dashboard is responsive.
Dashboard is accessible.
