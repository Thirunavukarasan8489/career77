1. Purpose

The Recruiter Analytics feature helps Recruiters understand recruitment performance and hiring activity.

Recruiters can view:

Job Performance
Application Trends
Candidate Pipeline
Interview Statistics
Hiring Conversion
Time to Hire
Recruitment Metrics
2. Analytics Access

Route:

/recruiter/analytics

Access requires:

Authenticated
+
Role = Recruiter
+
Account Status = Active
+
Verification Status = Approved
3. Analytics Flow
Recruiter
    ↓
Analytics
    ↓
Select Date Range
    ↓
Select Job / All Jobs
    ↓
View Recruitment Metrics
    ↓
Analyze Hiring Funnel
    ↓
Review Job Performance

9. Hiring Funnel

The Hiring Funnel shows Candidate progression.

Example:

Applications
    248
     ↓
Screening
     120
     ↓
Shortlisted
      42
     ↓
Interview
      24
     ↓
Offer
       8
     ↓
Hired
       5

This helps Recruiters identify where Candidates drop out of the recruitment process.

10. Candidate Conversion

Recruiters can view conversion rates.

Example:

Application → Shortlist
42 / 248

Shortlist → Interview
24 / 42

Interview → Offer
8 / 24

Offer → Hired
5 / 8

The system should calculate conversion percentages automatically.

11. Hiring Conversion

Example:

Hiring Conversion

Applications: 248
Hired: 5

Overall Hiring Conversion:
2.01%

The calculation should be based on the selected date range and Job filters.

12. Time to Hire

The system may display the average time required to hire Candidates.

Example:

Average Time to Hire

32 Days

Possible metrics:

Average Time to Hire
Fastest Hire
Longest Hire

The exact calculation should follow Career77's defined hiring date rules.

18. Recruitment Funnel Insights

Career77 may provide insights based on recruitment data.

Example:

Recruitment Insight

Your Full Stack Developer Job received
40% more Applications than last month.

Another example:

Recruitment Insight

A large number of Candidates are dropping
between Screening and Interview stages.

Insights should be presented as recommendations rather than definitive conclusions when data is limited.

While Analytics data is loading:

Analytics
    ↓
Loading
    ↓
Skeleton Cards
    ↓

    ↓

24. Error State

If Analytics cannot be loaded:

Unable to load analytics.

Please try again.

Action:

[ Retry ]

Internal system errors must not be exposed.

25. Data Accuracy

Analytics should use consistent data sources.

Example:

Applications
    ↓
Application Records

Pipeline
    ↓
Pipeline Records

Interviews
    ↓
Interview Records

Offers
    ↓
Offer Records

Hires
    ↓
Hiring Records

Metrics should be calculated using clearly defined rules.

26. Data Security

Recruiters must only view Analytics for Companies and Jobs they are authorized to manage.

Recruiter
    ↓
Company Authorization
    ↓
Job Authorization
    ↓
Analytics Access

Recruiters must not access another Company's Analytics.

28. Analytics Acceptance Criteria

The Analytics feature is complete when:

Recruiters can view recruitment metrics.
Recruiters can filter by date range.
Recruiters can filter by Job.
Application Trends are displayed.
Job Performance is displayed.
Hiring Funnel is displayed.
Candidate Conversion is displayed.
Hiring Conversion is displayed.
Time to Hire is displayed where data is available.
Time to Fill is displayed where data is available.
Interview Analytics are displayed.
Offer Analytics are displayed.
Recruitment Activity is displayed.
Top Performing Jobs are displayed.
Analytics Export is supported where implemented.
Empty states are handled.
Loading states are handled.
Error states are handled.
Analytics data is accurate.
Company-level authorization is enforced.
Job-level authorization is enforced.
Sensitive data is protected.
Analytics are responsive.
Analytics are accessible.
