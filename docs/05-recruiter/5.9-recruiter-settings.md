1. Purpose

The Recruiter Settings feature allows Recruiters to manage their account, security, notifications, privacy, and preferences.

Recruiters can manage:

Account Information
Password
Notifications
Privacy
Security
Preferences
2. Settings Access

Route:

/recruiter/settings

Access requires:

Authenticated
+
Role = Recruiter
+
Account Status = Active
+
Verification Status = Approved
3. Settings Structure
Recruiter Settings
│
├── Account
├── Security
├── Notifications
├── Privacy
└── Preferences
4. Account Settings

Recruiters can view and update basic account information.

Possible fields:

First Name
Last Name
Email
Phone
Profile Photo

Example:

Account Information

First Name: Gokulnath
Last Name: S
Email: recruiter@example.com
Phone: +91 XXXXX XXXXX

[ Save Changes ]
5. Email Address

The Recruiter can view their registered email address.

Changing the email address may require verification.

Flow:

Enter New Email
    ↓
Send Verification Code
    ↓
Verify Email
    ↓
Update Email

The system should not update the email until verification is completed.

6. Phone Number

Recruiters may add or update their phone number.

Flow:

Enter Phone Number
    ↓
Send OTP
    ↓
Verify OTP
    ↓
Update Phone Number

The system should validate the phone number before saving.

7. Profile Photo

Recruiters may upload or update their profile photo.

Supported formats may include:

PNG
JPG
JPEG
WEBP

The system should validate:

File Type
File Size
Image Dimensions
8. Password Settings

Recruiters can change their password.

Flow:

Current Password
    ↓
New Password
    ↓
Confirm New Password
    ↓
Validate
    ↓
Update Password

Password requirements should be clearly displayed.

Example:

Password must contain:

- Minimum required characters
- Uppercase letter
- Lowercase letter
- Number
- Special character

The exact password policy should follow Career77's security requirements.

9. Forgot Password

If a Recruiter forgets their password:

Forgot Password
    ↓
Enter Email
    ↓
Receive Reset Link / OTP
    ↓
Verify Identity
    ↓
Create New Password
    ↓
Login

Password reset links should expire after a defined period.

10. Active Sessions

If Career77 supports session management, Recruiters may view active sessions.

Example:

Active Sessions

Chrome — Windows
Current Session

Chrome — Android
Last Active: 2 hours ago

[ Sign Out ]

Recruiters may sign out from individual sessions.

11. Two-Factor Authentication

If supported, Recruiters can enable Two-Factor Authentication.

Flow:

Enable 2FA
    ↓
Verify Identity
    ↓
Configure Authentication Method
    ↓
Verify Code
    ↓
2FA Enabled

Possible methods:

Authenticator App
Email OTP
SMS OTP

The available methods should follow Career77's security architecture.

12. Notification Settings

Recruiters can manage notification preferences.

Possible notification categories:

New Applications
Candidate Messages
Interview Updates
Interview Reminders
Job Status Updates
Company Verification
System Notifications

Example:

Notification Preferences

New Applications       [ ON ]
Interview Updates      [ ON ]
Interview Reminders    [ ON ]
Candidate Messages     [ ON ]
Job Status Updates     [ ON ]
13. Notification Channels

Recruiters may control notification channels.

In-App
Email

Example:

New Application

In-App    [ ON ]
Email     [ ON ]

The system should respect notification preferences.

14. Privacy Settings

Recruiters can manage applicable privacy settings.

Possible options:

Profile Visibility
Contact Information Visibility
Activity Visibility

Private Recruiter information should not be publicly displayed unless explicitly allowed.

15. Account Security

Security settings may include:

Password
Two-Factor Authentication
Active Sessions
Login Activity

Recruiters should be able to review important security activity.

Example:

Recent Login

Device: Chrome / Windows
Location: India
Date: 22 July 2026
16. Login Activity

If supported, the system may display recent login activity.

Information may include:

Device
Browser
Approximate Location
Date
Time

Sensitive security information should be protected.

17. Account Deactivation

Recruiters may request account deactivation.

Flow:

Settings
    ↓
Deactivate Account
    ↓
Confirmation
    ↓
Optional Reason
    ↓
Confirm
    ↓
Account Deactivated

Example:

Are you sure you want to deactivate your account?

[ Cancel ]
[ Deactivate Account ]

The system should clearly explain the impact before deactivation.

18. Account Deletion

If Career77 supports permanent account deletion:

Settings
    ↓
Delete Account
    ↓
Confirm Identity
    ↓
Confirm Deletion
    ↓
Account Deletion Request

Deletion must follow Career77's data retention, legal, and audit policies.

19. Logout

Recruiters can log out from the Recruiter Portal.

Flow:

Recruiter Portal
    ↓
Logout
    ↓
Session Terminated
    ↓
Login Page

The system should invalidate the appropriate session or authentication token.

20. Settings Validation

The system should validate all updated information.

Examples:

Email address is invalid.
Phone number is invalid.
Password does not meet requirements.
Passwords do not match.

The Recruiter should be able to correct errors before saving.

21. Save Changes

After modifying settings:

Edit Settings
    ↓
Update Information
    ↓
Validation
    ↓
Save Changes
    ↓
Success Message

Example:

Your settings have been updated successfully.
22. Error Handling

If an update fails:

Unable to update your settings.

Please try again.

Action:

[ Retry ]

Internal API or database errors must not be exposed.

23. Settings Security

The Settings feature must:

Require authentication.
Verify Recruiter role.
Verify account status.
Protect account information.
Secure password changes.
Require verification for sensitive changes.
Protect authentication sessions.
Enforce backend authorization.
Record important security actions.
24. Sensitive Actions

The system may require additional authentication for:

Change Password
Change Email
Enable / Disable 2FA
Deactivate Account
Delete Account

Possible verification methods:

Current Password
OTP
Email Verification
2FA Code
25. Audit Logs

Important Settings actions should be recorded.

Examples:

Password Changed
Email Changed
Phone Changed
2FA Enabled
2FA Disabled
Session Revoked
Account Deactivated
Account Deletion Requested

Audit records should include:

Action
User
Date
Time
26. Empty State

If no additional settings are configured:

No additional preferences configured.

Configure your settings to personalize
your Career77 Recruiter experience.
27. Loading State

While Settings are loading:

Settings
    ↓
Loading
    ↓
Skeleton Settings Sections
    ↓
Settings Loaded
28. Responsive Design

The Settings feature must support:

Desktop
Tablet
Mobile

Desktop:

Settings Navigation
    +
Settings Content

Mobile:

Settings Sections
    ↓
Selected Setting
    ↓
Settings Form

Forms should be easy to use on smaller screens.
